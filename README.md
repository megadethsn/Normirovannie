# Генератор нормированных заданий

Desktop-приложение для формирования рабочих файлов `.docx` и `.pdf` по шаблонам Word.

## Запуск из исходников

```bash
pip install -r requirements.txt
python main.py
```

## Шаблоны без пересборки

Приложение всегда сначала ищет шаблоны в папке `templates`, которая лежит рядом с `main.py` или рядом с собранным `.exe`.

Чтобы изменить итоговый документ без пересборки приложения:

1. Откройте папку `templates` рядом с `.exe`.
2. Замените нужный `.docx`-шаблон файлом с тем же именем.
3. Запустите приложение заново и сформируйте документ.

При первом запуске собранного `.exe` папка `templates` будет создана автоматически, если ее еще нет.

## Центры и сотрудники

При первом запуске приложение создает файл `centers.json` рядом с `.exe` или рядом с исходниками. В нем хранятся центры, сотрудники, имена шаблонов и признак дальневосточного центра.

Менять эти данные можно через кнопку `Центры` на главной странице приложения. После сохранения страницы центров будут перестроены без пересборки.

На странице `Центры` можно выбрать шаблоны через проводник. Выбранный `.docx` должен лежать в папке `templates`.

## УИНы

Поле УИН поддерживает одиночные значения и диапазоны:

```text
3580
3580-3585
п3580-п3585
g3580-g3585
3580-3582, п4000
```

Обычные УИНы получают префикс `11...`, а значения с `п` или `g` получают префикс `12...`.

## Логи

Диагностика работы приложения записывается в файл `app.log` рядом с `.exe` или рядом с исходниками. Если PDF не сформировался или Word вернул ошибку, сначала проверьте этот файл.
